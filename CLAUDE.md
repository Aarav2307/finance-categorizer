# Finance Categorizer

## Design Context

This project has a defined design system. Before making frontend changes:

- **[PRODUCT.md](PRODUCT.md)** — who this is for (a single user doing a monthly deep review), what it's for, brand personality ("calm, precise, trustworthy"), and anti-references (no generic SaaS dashboard chrome, no gamified consumer-finance energy).
- **[DESIGN.md](DESIGN.md)** — the visual system ("The Quiet Ledger"): a near-black tonal stack, one muted-violet accent (#7c6fda) carrying every interface signal, and named rules like *The One Signal Rule* and *The Tonal Depth Rule*.

DESIGN.md wins on visual decisions; PRODUCT.md wins on strategic/voice decisions. Read both before designing or critiquing any surface.
